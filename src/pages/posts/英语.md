英语太重要了，想翻阅的各种文档，各种很新的技术，其网站无一例外全是英文。

当我想象到这样一幅画面：国外程序员想弄点好玩的，使用某语言开始写，遇到了一些困难，直接去官方文档翻阅资料，看了一会遍豁然开朗，于是一气呵成写完了一个功能，整个流畅完美又通畅，而我在想写什么功能后，翻阅官方文档，使用我那令人堪忧的英语水平读了十分钟读完了几句话，而且还是不甚明白。

前几天在想写一个自动生成 url 的小功能，本想直接字符串拼接，Daniel 看到了直接指出该语言有直接生成 url 的 api ，他带着我读了文档中描写 api 的段落，并带着我完成了该功能，而我看他翻阅文档，真是太羡慕这种英语水平了。

刚刚无端联想了一下，我这种人放在国外，那岂不是就是 “字没识全的小孩子在写文章” 一样的感觉。

高中时期听说编程需要英文好，到了大学跟着课程写一些小算法，又觉得英文不是那么重要，就是那么几个关键字，然后自己写程序就好了，到现在又发现，编程需要英语好的真正意思是当自己想实现什么功能的时候，想查阅文档，查询 api 的使用方法时，能够通顺的阅读文档，并理解其中的意义。

但有人说，使用翻译器也是可以的呀，我觉得可以是可以，但他翻译的不是完全正确的，翻译器给一些术语，或者是段落中有一些代码或者其他的不需要翻译的词语，全部翻译，就是“机翻”，总会偏离一些意思并且有些地方可能并不会读的很通顺，于我来说，就是用起来“很难受”，但由于我英语水平实在不咋地，所以我也经常使用翻译器一段一段的翻译。

最近一直在尝试直接阅读英文文档，遇到不会的词就翻译一下，然后自己理解文档的意思，希望慢慢能够提升一下我的英语水平。

什么时候我的英语水平也能提高到能流利通顺的阅读官方文档啊😭！
